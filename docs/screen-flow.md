# 画面遷移マップ（UX再設計）

このドキュメントは、MVP優先で実装可能なUX方針を含む遷移設計です。

## 前提

- 対象: iOS / Android（Expo + React Native）
- タブ構成: `Home` と `Settings` の2タブ
- 未ログイン時は認証ハブを表示

## 画面一覧

1. `認証ハブ`
- ルート案: `/auth`
- 役割: `ログイン` / `アカウント作成` の入口

2. `ログイン`
- ルート案: `/auth/sign-in`
- 役割: `username` / `token` 入力で利用開始

3. `アカウント作成`
- ルート案: `/auth/sign-up`
- 役割: Pixelaユーザー作成

4. `Homeタブ`
- ルート案: `/home`
- 役割: グラフカード一覧、最短記録入力、詳細画面への導線

5. `ピクセルクイック追加シート`
- ルート案: Home内の Bottom Sheet
- 役割: 当日中心の素早い記録追加

6. `Settingsタブ`
- ルート案: `/settings`
- 役割: ユーザー設定、外部リンク、危険操作

7. `グラフ詳細画面`
- ルート案: `/graphs/[graphId]`
- 役割: Month/Year単位での記録確認、軽量統計、期間内一覧表示、記録詳細への入口

8. `グラフ関連Stackナビゲーション`
- ルート案: `/graphs/*`
- 役割: Graph/Pixel系画面のヘッダーと戻る導線をネイティブStackで統一

補足:
- 記録一覧専用画面（`/graphs/[graphId]/pixels`）は持たず、Graph詳細の期間内一覧を唯一の記録参照導線とする

## HomeタブのUI方針

- Home/SettingsはExpo標準ヘッダーを使用する（画面内ヘッダーの重複表示はしない）
- Homeヒートマップは直近14週の固定表示（モード切替なし）
- Homeカードの導線は `セルタップ=日付指定で記録` / `+ = 今日を記録` / `カードタップ=Detail` に固定する
- Homeカードから `記録一覧 / 統計 / 編集 / 削除 / ...` は表示しない
- `グラフ追加` はHomeヘッダー右上アクションから遷移する
- pull-to-refresh を提供
- グラフの作成・編集ではカラーテーマを選択し、カードUIとヒートマップ色へ反映する
- HomeのTodayエリアは固定表示にせず、グラフ一覧と同じスクロール領域の先頭で表示する
- Todayエリアでは未入力グラフを上位1件のみ表示し、必要に応じて `他X件未入力` の補助表示を行う

## Pixel追加UX方針

1. 標準操作:
- Homeカードの `+` またはヒートマップセルタップで Bottom Sheet を開く
- 初期値は「今日」、入力項目は `date` / `quantity` / 任意メモ（`optionalData`）
- 保存成功時はシートを閉じ、カード内容を更新し、Home上部にToastコンポーネントで通知する

2. 補足:
- 記録追加導線はHome内Bottom Sheetに一本化し、専用の詳細入力画面は設けない

3. 削除など危険操作:
- すべて確認ダイアログ必須

## 遷移フロー（推奨）

1. 未ログイン時は `認証ハブ` 表示
2. `ログイン` または `アカウント作成` 成功で `Home` へ
3. Homeで `+` またはヒートマップセルを押下
4. Bottom Sheet でクイック追加
5. Homeカードのカードタップで `グラフ詳細画面` へ遷移
6. 詳細画面で `Month/Year` を切り替えて期間別に確認
7. 詳細画面の記録一覧行タップで `記録編集` (`/graphs/[graphId]/pixels/[date]`) へ遷移
8. `Settings` からプロフィール/規約/ログアウト等へ遷移

## 遷移実装ルール（実装確定）

- 通常導線（画面を積み上げる遷移）は `push` / `back` を使う
- セッション境界（ログイン完了、ログアウト、認証欠落リダイレクト）は `replace` を使う
- 本フェーズでは Stack の `animation` 種別は変更しない

## ヒートマップ方針

- MVP:
  - Home: 自前ヒートマップ（14週=98日、5段階: 薄灰+テーマ色4段階）
  - Homeヒートマップの描画データはグラフカード単位で `GET /v1/users/{username}/graphs/{graphID}/pixels` を取得する
  - Homeヒートマップは GitHub 風に、上部へ月境界ラベル・左側へ `Mon / Wed / Fri` ラベルを表示する
- MVP後:
  - 詳細画面の `Year` 表示を含めて自前描画を継続改善
- カラーテーマ方針:
  - Pixelaの `shibafu / momiji / sora / ichou / ajisai / kuro` をUIテーマ候補として表示
  - 選択した色は、グラフカードのアクセント色とヒートマップ描画色に統一して反映する

## 更新ルール

- 画面追加・削除・名称変更時は本ファイルを同コミットで更新する
- UX方針変更時は `HomeタブのUI方針` と `Pixel追加UX方針` を同時更新する
- Graph/Pixel系ルートのヘッダー/戻る導線を変更した場合は、Stack方針を同コミットで更新する

## 共通UI適用メモ

- `auth` / `settings` / `graphs` の主要画面は、`src/shared/ui/` の共通コンポーネントへ段階適用する
- 共通化後も、本ドキュメントで定義した画面導線と遷移順は変更しない
- テスト導入（Jest/RNTL/Maestro）は品質担保を目的とし、画面遷移そのものは変更しない

## Detail期間の暫定定義

- Graph詳細の `Month` は「暦月」を意味する
- Graph詳細の `Year` は「暦年」を意味する
- Homeの14週固定表示とは役割を分離し、同一期間を示すものではない
- 詳細画面は期間内の記録一覧とLight統計（合計・記録日数・最大値）を表示する
- 記録一覧は `optionalData` がある行のみ要約を1行表示する
- 画面上には補助文言として `Month=暦月 / Year=暦年` を表示し、Homeの14週固定表示と誤解しないようにする
