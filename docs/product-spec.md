# アプリ設計ドキュメント

## 1. プロダクト概要

このアプリは **Pixela API をベースにしたクライアントアプリ** です。  
ただの API 操作UIではなく、 **GitHub contribution のようなヒートマップ体験** を通じて

- 自分の行動の可視化
- 緑が増える喜びによるモチベーション維持
- 過去記録の蓄積・参照

を提供することを主目的としています。

> 分析や傾向解析は優先度が低く、視覚体験・入力体験を重視します。

---

## 2. 設計思想

### 2.1 主な UX の価値

- ヒートマップの「面積」と「色の濃淡」による視覚的な満足感
- 今日の入力を最短で行えるインタラクション
- Pixela の概念を尊重しつつ、一般ユーザーにも追体験できる UI

### 2.2 Pixela 由来の概念

- 色は Pixela の色設定をそのまま使用
- 「compact / full」といった Pixela 固有の用語は UI に出さない
- 期間は論理的に意味のある単位で表現（例：直近 14 週間）

---

## 3. 画面設計

### 3.1 Home

#### 目的

- ヒートマップを主役にした視覚体験
- 今日の入力を最短で可能にする
- 詳細な統計に誘導せず、まずは貢献の実感を提供

#### 構成

1. **App Bar**
   - 期間表示例：`Nov – Feb` または `Last 14 weeks`
   - グラフ追加ボタン `+`
   - iOS/AndroidではExpo標準ヘッダーを優先利用する

2. **Today エリア**
   - 今日の日付
   - 未入力のグラフを優先して表示（上位1件のみ）
   - 未入力が複数ある場合は補助表示（例: `他X件未入力`）を表示可能
   - タップ → 即入力モーダル
   - Home全体の情報密度を保つため、視覚トーンは控えめに保つ
   - 固定表示にはせず、グラフ一覧と同じ縦スクロール領域内に配置する

3. **Graph カード**
   - カードヘッダー
     - グラフ名
     - 今日の入力値（例：`Today 20min` or `—`）
     - 右上 `+`（今日入力の最短導線）
   - ヒートマップ本体
     - 7×14 の固定サイズ
     - Pixela の色設定をそのまま使用
   - カードフッター（補助情報）
     - 単位（例：`min`）
     - 今月合計（任意）

#### インタラクション

- **セルタップ** → 入力モーダル
- **`+` タップ** → 今日を初期値にした入力モーダル
- **カードタップ** → 詳細画面

備考：横スクロール不要。縦スクロールのみ。

---

### 3.2 Graph 詳細画面

#### 目的

- 過去データの確認
- 軽い統計の表示
- 管理操作（編集/削除）の集約

#### 構成

1. グラフ名
2. 期間切り替え
   - **Month**
   - **Year**
     - 年ビュー（full 相当）
   - 補助文言として `Month=暦月 / Year=暦年` を常時表示し、Homeの14週固定表示と区別する
3. Light 統計
   - 合計値
   - 記録日数
   - 最大値
4. 記録一覧
   - 日付・数値
   - 行タップで記録詳細（更新/削除）へ遷移
5. `...` メニュー
   - 編集導線
   - 削除導線（確認ダイアログ必須）

※ 分析チャート（折れ線等）は今の設計では不要とする。

---

## 4. 入力体験設計

### 4.1 入力モーダル

- 数値入力（テキスト or ステッパー）
- 前回値をサジェスト表示
- `+1`, `+5` ボタン（任意）
- 保存は最短動作
- 入力可能な `quantity` は **1以上**（0は入力不可）
- 未入力は「当日レコードが存在しない」場合のみ
- `quantity <= 0` を受信した場合は防御的に未入力相当として扱う（仕様上は生成しない）

---

## 5. 表示ルール

### 5.1 カラー

- Pixela の色設定をそのまま使用
- **未入力セルはグレーで表現**

### 5.2 期間表示

- Home は直近 14 週間（固定）
- 詳細は Month / Year 表示（それぞれ暦月 / 暦年）

---

## 6. UX の優先度

高 → 低

緑の広がりの視覚体験
↓
今日の入力のしやすさ
↓
過去の記録参照
↓
軽い統計
↓
高度な分析（現状不要）

## 8. 要点まとめ

| 概念        | 方針                        |
| ----------- | --------------------------- |
| 色          | Pixela カラーをそのまま使用 |
| 期間        | Home は直近 14 週間         |
| Home UI     | 横スクロール不要            |
| 入力UX      | セルタップで即編集          |
| 統計        | 詳細画面に移譲              |
| Pixela 用語 | UI では極力出さない         |
