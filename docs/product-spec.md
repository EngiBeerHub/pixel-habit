# アプリ設計ドキュメント

## 1. プロダクト概要

このアプリは **Pixela API をベースにしたクライアントアプリ** です。  
ただの API 操作UIではなく、 **GitHub contribution のようなヒートマップ体験** を通じて

- 自分の行動の可視化
- 緑が増える喜びによるモチベーション維持
- 過去記録の蓄積・参照

を提供することを主目的としています。

> 分析や傾向解析は優先度が低く、視覚体験・入力体験を重視します。

---

## 2. 設計思想

### 2.1 主な UX の価値

- ヒートマップの「面積」と「色の濃淡」による視覚的な満足感
- 今日の入力を最短で行えるインタラクション
- Pixela の概念を尊重しつつ、一般ユーザーにも追体験できる UI

### 2.2 Pixela 由来の概念

- 色は Pixela の色設定をそのまま使用
- 「compact / full」といった Pixela 固有の用語は UI に出さない
- 期間は論理的に意味のある単位で表現（例：直近 14 週間）

---

## 3. 画面設計

### 3.1 Home

#### 目的

- ヒートマップを主役にした視覚体験
- 今日の入力を最短で可能にする
- 詳細な統計に誘導せず、まずは貢献の実感を提供

#### 構成

1. **App Bar**
   - タブ表示名は `Habits`（内部ルートは `/home` を継続）
   - 右上にグラフ追加アイコンボタン（ヘッダー標準ボタンの見た目を優先）
   - iOS は標準Large Titleヘッダーを採用し、スクロールで通常タイトルへ縮退する
   - iOS/AndroidではExpo標準ヘッダーを優先利用する

2. **Graph カード**
   - カードヘッダー
     - グラフ名
     - 右上 `+`（今日入力の最短導線）
   - ヒートマップ本体
     - 7×14 の固定サイズ
     - Pixela の色設定をそのまま使用
   - 補助情報は常設しない（ID/単位/タイムゾーンなどの詳細はGraph詳細画面へ集約）

#### インタラクション

- **セルタップ** → 入力モーダル
- **`+` タップ** → 今日を初期値にした入力モーダル
- **カードタップ** → 詳細画面
- 未来日セルは非活性とし、カードタップ導線へ遷移させない

備考：横スクロール不要。縦スクロールのみ。

---

### 3.2 Graph 詳細画面

#### 目的

- 過去データの確認
- 軽い統計の表示
- 管理操作（編集/削除）の集約

#### 構成

1. ヘッダー
   - iOSはLarge Titleヘッダーでグラフ名を表示する
   - 画面内に同名の見出しを重複表示しない
2. 期間切り替え
   - **Month**
   - **Year**
     - 年ビュー（full 相当）
   - 補助文言として `Month=暦月 / Year=暦年` を常時表示し、Homeの14週固定表示と区別する
3. 統計
   - 合計値
   - 記録日数
   - 最大値
   - 平均（記録日）
   - 現在連続日数
   - 最長連続日数
4. 記録一覧
   - `yyyy/MM/dd (EEE)` の日付・数値
   - メモ（optionalData）がある場合のみ、先頭要約を1行表示
   - 行タップで記録詳細（更新/削除）へ遷移
   - 記録一覧専用画面は設けず、Graph詳細画面内で完結する
5. 管理操作
   - 編集アイコン導線
   - 削除アイコン導線（確認ダイアログ必須）

※ 分析チャート（折れ線等）は今の設計では不要とする。

---

## 4. 入力体験設計

### 4.1 入力モーダル

- 数値入力（テキスト or ステッパー）
- 前回値をサジェスト表示
- `+1`, `+5` ボタン（任意）
- 保存は最短動作
- 記録追加導線はHome内モーダル（Bottom Sheet）に一本化し、専用の詳細入力画面は設けない
- 入力可能な `quantity` は **1以上**（0は入力不可）
- `optionalData` は任意の補足メモとして入力可能
- 保存成功時はToast + 主要操作の触覚フィードバックで完了を返す
- 未入力は「当日レコードが存在しない」場合のみ
- `quantity <= 0` を受信した場合は防御的に未入力相当として扱う（仕様上は生成しない）
- Homeの情報密度維持のため、メモ表示はGraph詳細の記録一覧に限定し、Homeカードには表示しない

## 5. 表示ルール

### 5.1 カラー

- Pixela の色設定をそのまま使用
- **未入力セルはグレーで表現**

### 5.2 期間表示

- Home は直近 14 週間（固定）を描画に使うが、ヘッダー下へ期間テキストは常設しない
- 詳細は Month / Year 表示（それぞれ暦月 / 暦年）

---

## 5.3 Navigation Policy（iOS / Large Title 運用ルール）

### 目的

本アプリにおける iOS ナビゲーション設計の一貫性を保つため、Large Title の使用基準を明文化する。  
Large Title は「画面の文脈（セクション）を強調するための視覚装置」であり、常に使うものではない。

### 基本原則

1. **Large Title は「閲覧・探索」文脈で使う**
   - 情報を縦スクロールで俯瞰するページ
   - 一覧・サマリー・ダッシュボード
   - タブのトップ画面
   - オブジェクトのトップページ（閲覧中心）

2. **入力・編集中心画面では使わない**
   - フォーム入力画面
   - 編集画面
   - 作成画面
   - 確定操作が主目的の画面
   - 短くスクロールしない画面
   - これらは通常タイトルを使用する

### 本アプリへの適用

- **Habits（タブトップ）**
  - ヒートマップ一覧
  - スクロールで俯瞰
  - → Large Title: 使用

- **Graph Detail（Habit トップページ）**
  - Month/Year 切替
  - 統計
  - 期間内記録一覧
  - 情報閲覧が主目的
  - → Large Title: 使用可（推奨）

  注意:
  - ヘッダーとコンテンツ内に同一タイトルを重複表示しない
  - タイトルはヘッダー側に寄せる

- **記録編集**
  - 入力フォーム中心
  - → Large Title: 不使用

- **トークン変更**
  - 設定フォーム
  - → Large Title: 不使用

- **Settings（トップ）**
  - 設定項目一覧
  - → Large Title: 使用可

### Screen 実装方針（expo-router / Stack）

- `headerLargeTitle` は Screen 単位で制御する
- Stack 全体に一律適用しない
- 一覧系は ON、フォーム系は OFF

### 禁止事項

- すべての画面で Large Title を使うこと
- フォーム画面で Large Title を使うこと
- 同一タイトルをヘッダーと本文に重複表示すること

### デザイン一貫性

- Large Title = セクションの入口
- 通常タイトル = 操作画面
- モーダル画面は原則通常タイトル

## 6. UX の優先度

高 → 低

緑の広がりの視覚体験
↓
今日の入力のしやすさ
↓
過去の記録参照
↓
軽い統計
↓
高度な分析（現状不要）
