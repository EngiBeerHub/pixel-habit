# MVP Definition

このドキュメントは「最初に作る範囲」を固定して、手戻りを減らすためのものです。
重くしないために、機能は最大3つまでに絞ります。

## Goal

- Pixela API を使った日次記録アプリの最小価値を、Expo + React Native で動かす
- UI は `heroui native` + `uniwind` を前提とする
- 既存 Ionic 版は参照のみ。実装は RN 向けに再設計する

## Out of Scope (MVPではやらない)

- 通知、ウィジェット、オフライン同期、複雑な分析画面
- 多言語対応
- 高度な設定画面

## Prioritized Features (最大3つ)

### 1. ログイン/接続設定（Pixela接続情報の保存）

- 目的:
  - ユーザーが Pixela API を叩ける状態を作る
- 入力:
  - `username`
  - `token`
- 受け入れ条件:
  - 入力値の必須チェックがある
  - 保存後に再起動しても値が保持される（`expo-secure-store`）
  - 不正値の場合にエラーメッセージが表示される

### 2. グラフ一覧表示

- 目的:
  - 対象ユーザーが利用可能なグラフを確認できる
- 受け入れ条件:
  - 初回表示時にAPIから一覧取得できる
  - ローディング/エラー/空状態のUIがある
  - 再取得（pull-to-refresh or 再試行）ができる

### 3. 日次記録の追加

- 目的:
  - 任意グラフへ当日の記録を追加できる
- 受け入れ条件:
  - 数値入力のバリデーションがある
  - 保存成功時に完了フィードバックが表示される
  - 保存失敗時に理由を表示し、再試行できる

## First Vertical Slice

最初に「1本通す」フロー:

1. 接続設定入力
2. 保存
3. グラフ一覧取得
4. 1件選択して日次記録追加

この流れが通ったら、次の機能追加に進む。

## Backlog (MVP後)

- グラフ作成/編集/削除
- 記録更新/削除
- カレンダー表示
- UIテーマ調整
