# UI / UX Rules

## Purpose

画面設計と実装時に、説明文の混入や情報重複によるノイズを防ぎ、ネイティブアプリらしい情報密度を保つ。

## Production Copy Only

- UI に置く文言は、本番アプリでそのまま表示してよい固定ラベルだけにする
- 画面説明、意図説明、操作説明、要約文を、補助テキストとして安易に追加しない
- デザインレビュー中の考えや会話上の説明を、そのまま UI 文言へ変換しない
- 新しい文言を足す前に「既存のネイティブアプリ UI に本当にあるか」で確認する

## Avoid Redundant Information

- 同じ意味の情報は、1 画面内で 1 回だけ見せる
- Graph 名、件数、並び順、対象範囲などは、近接するセクションで重複表示しない
- 追加する情報は「新しい判断材料を増やすか」で評価し、増やさないなら置かない

## CTA And Navigation

- 遷移導線の役割しかない要素には、疑似フィルタや疑似設定値を載せない
- CTA カードや行は、遷移先が明確に分かる最小限の情報に留める
- 遷移先画面で扱う情報は、元画面で先回りして説明しない

## Header Rules

- ネイティブナビゲーション配下の画面では、large title を無条件に使わない
- 画面の主語が graph で明確な場合、ヘッダーと直下セクションで graph 名を重複表示しない
- ヘッダーに主情報を置いた場合、その直下では補助情報だけを扱う

## Review Checklist

- この文言は本番 UI として自然か
- 同じ情報を別の場所でも出していないか
- その補助テキストがなくても意味が通るか
- それは説明文ではなく、実際の UI ラベルか
- 遷移導線に不要な属性や状態を載せていないか
