# Implementation Plan

- [x] 1. Native Tabs基盤を確定し、タブ遷移の標準挙動を固定する
- [x] 1.1 Habits/Settings の主導線を Native Tabs 前提で統一し、タブ表示要素の一貫性を確保する
  - タブ基盤を Native Tabs として固定し、タブ名・アイコン・選択状態の表示差をなくす。
  - タブ切替後に直前の文脈へ戻れる操作感を維持し、ユーザーが現在地を見失わないことを確認する。
  - _Requirements: 1.1, 1.3, 1.5_
- [x] 1.2 タブ切替時のコンテキスト保持と失敗時リカバリ導線を定義する
  - タブ切替失敗時に現在タブへ留まり、再試行可能な通知を返す挙動を定義する。
  - タブ内の遷移履歴が不意に破棄されないことを検証する。
  - _Requirements: 1.2, 1.4_

- [ ] 2. Large Titleポリシーを画面種別ごとに適用する
- [ ] 2.1 閲覧画面と操作画面のタイトルモードを分類し、画面単位で適用する
  - 一覧・俯瞰画面は Large Title、フォーム画面は通常タイトルを適用する。
  - 全体一律設定を避け、画面ごとの責務に沿ったタイトル運用へ統一する。
  - _Requirements: 2.1, 2.2, 2.3_
- [ ] 2.2 ヘッダーと本文の同一タイトル重複を解消する
  - ヘッダー側でタイトルを担保する画面では本文の重複見出しを排除する。
  - 視認性と情報階層の一貫性を崩さない表示構成へ整理する。
  - _Requirements: 2.4_

- [ ] 3. 標準バック導線とヘッダー右上アクションを標準化する
- [ ] 3.1 画面階層に応じた戻る操作をOS標準に統一する
  - スタック先頭以外では標準バックを表示し、先頭画面では疑似戻るを表示しない。
  - プラットフォーム既定の遷移アニメーションを維持し、見た目だけの模倣実装を排除する。
  - _Requirements: 3.1, 3.2_
- [ ] 3.2 Graph Detail の `...` 操作をネイティブメニューで提供し、視覚整合を維持する
  - 右上メニューの編集/削除導線をネイティブメニュー経由で提供する。
  - アイコン配置・タップ領域・色トーンをヘッダーアクション規約に揃える。
  - _Requirements: 3.3, 3.4_
- [ ] 3.3 ネイティブメニュー制約時のフォールバック導線を整備する
  - ネイティブメニュー機能が使えない実行環境でも編集/削除が失われない導線を用意する。
  - プラットフォーム差異と既知制約を運用ドキュメントへ明記する。
  - _Requirements: 5.2, 5.3_

- [ ] 4. Habitsの主要操作導線を安定化する
- [ ] 4.1 Habitsヘッダーの単一追加導線を維持し、カード操作遷移を保証する
  - ヘッダーの追加入口を一つに固定し、誤操作を減らす。
  - カードタップ・セルタップ・追加操作が意図しない画面遷移を起こさないことを担保する。
  - _Requirements: 4.1, 4.2_
- [ ] 4.2 未来日セル保護とロード状態表示を統一する
  - 未来日セルの押下時に不要遷移を発生させない。
  - データ取得中表示をナビ体験と矛盾しない形で統一する。
  - _Requirements: 4.3, 4.4_

- [ ] 5. ナビゲーション品質をテストで固定する
- [ ] 5.1 (P) タブ切替・戻る導線・画面遷移の統合テストを拡充する
  - タブ移動、スタック遷移、戻る導線が仕様通りに動く回帰テストを整備する。
  - iOS/Android で同等の遷移結果になることを検証観点として固定する。
  - _Requirements: 5.1, 6.1_
- [ ] 5.2 (P) ヘッダーメニューの起動・アクション分岐・フォールバック挙動を検証する
  - メニュー起動、編集/削除遷移、ネイティブ不可時の代替導線をテストで担保する。
  - ヘッダーアクション変更時の回帰検知が可能な粒度へ整理する。
  - _Requirements: 3.3, 5.2, 6.2_
- [ ] 5.3 Requirement ID とテストケース対応表を仕様成果物に固定する
  - 要件ID単位でテスト責務を追跡可能にし、将来変更時の抜け漏れを防ぐ。
  - _Requirements: 6.3_

- [ ] 6. 最終統合検証と品質ゲートを完了する
- [ ] 6.1 iOS/Android/制約環境を含む手動スモークでナビ一貫性を確認する
  - 標準バック、Large Title縮退、ヘッダーアクション、タブ文脈保持を通し確認する。
  - _Requirements: 1.2, 3.1, 5.1_
- [ ] 6.2 品質ゲートを実行し、差分をグリーンで確定する
  - 自動テスト・静的チェック・型検査を通過させる。
  - _Requirements: 6.1, 6.2, 6.3_
